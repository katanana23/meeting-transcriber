# Инструкции для Claude Code

Этот репозиторий — каркас, написанный без доступа к macOS. Твоя задача — довести его до рабочего состояния на конкретной машине (MacBook M4, AirPods, BlackHole 2ch установлен).

## Порядок работы — строго по одному шагу, с проверкой после каждого

1. `bun install` → `cargo check` в src-tauri. Почини ошибки компиляции, не меняя архитектуру.
2. Запусти `bun tauri dev`. Убедись что окно открывается и `list_devices` возвращает реальные устройства.
3. Протестируй запись ТОЛЬКО с микрофона (закомментируй sys-поток временно). Проверь что WAV играется и содержит голос.
4. Подключи BlackHole-поток. Проверь запись системного звука (включи любое видео на YouTube).
5. Проверь whisper-cli на записанном WAV вручную из терминала, потом через приложение.
6. Проверь полный цикл: запись → стоп → транскрипт на экране → .md файл в vault.

## Известные слабые места (чинить в первую очередь)

- `audio.rs`: глобальный реестр WRITERS — упрощение. Если будут проблемы с финализацией WAV, перенеси writer'ы внутрь структуры Recorder.
- `audio.rs`: sample rate у AirPods-микрофона может быть 16/24kHz, у BlackHole 48kHz — whisper-cli сам ресемплирует, но проверь что WAV-заголовки корректны.
- `whisper.rs`: путь к бинарю — предполагается `whisper-cli` в PATH (brew). Если у пользователя иначе, добавь настройку пути.
- `lib.rs`: `stop_and_transcribe` блокирует поток — для длинных записей оберни в `tauri::async_runtime::spawn_blocking` и добавь событие прогресса.
- Tauri 2 Info.plist: убедись что NSMicrophoneUsageDescription попадает в бандл (может потребоваться настройка в tauri.conf.json bundle > macOS > infoPlist).
